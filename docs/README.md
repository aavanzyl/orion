# Documentation

Detailed guides for Orion — an AI workflow orchestrator that drives coding agents through a
deterministic DAG of steps, tracked live on a Kanban board.

New here? Start with [Getting Started](./getting-started.md), then skim
[Architecture](./architecture.md) to understand how the pieces fit together.

| Guide | What's inside |
| --- | --- |

| [Architecture](./architecture.md) | Components, the adapter model, and how a run flows end to end |
| [Configuration](./configuration.md) | The full `.orion/config.yaml` schema and command templates |
| [Workflows](./workflows.md) | Every node type, retries, parallel fan-out, matrix, loops, conditions, sub-workflows |
| [Skills](./skills.md) | The skill catalog, materialization, and installing from GitHub |
| [Adapters](./adapters.md) | Harness, SCM, board, and communication providers |
| [Integrations](./integrations.md) | Codebase RAG, MCP servers, chat, schedules, and board sync |
| [API Reference](./api.md) | REST and SSE endpoints |

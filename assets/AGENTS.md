# Orion Brand & Media Guide

This document defines the visual language for Orion.

Every image, illustration, diagram, screenshot, banner, logo or icon generated for this repository should follow these guidelines.

---

# Brand Personality

Orion is **not** another AI chatbot.

It is a deterministic workflow engine that orchestrates autonomous coding agents.

The workflow owns the process.
The agent owns the reasoning.

The product should feel:

- Calm
- Intelligent
- Premium
- Technical
- Approachable
- Optimistic

Never feel:

- Corporate
- Cyberpunk
- Hacker
- Military
- Dark evil AI
- Generic SaaS
- Purple AI blob

---

# Core Theme

Everything revolves around the **Orion constellation**.

The constellation is the visual metaphor for a workflow DAG.

Stars = Nodes

Connections = Dependencies

The constellation should appear throughout branding whenever possible.

Never draw random stars.

Always reference the actual Orion constellation.

---

# Visual Style

Preferred style:

- clean vector art
- soft anime inspiration
- minimalist
- subtle glow
- rounded shapes
- high contrast
- premium gradients
- cinematic lighting
- soft atmosphere

Think:

Apple × GitHub × Makoto Shinkai

NOT

Marvel
Cyberpunk 2077
Blade Runner
Hackerman

---

Primary

#a855f7

#38bdf8

#0B1020

Accent

White

---

# Lighting

Everything softly glows.

No harsh bloom.

No lens flares.

No excessive particles.

Stars should feel magical but restrained.

---

# Shapes

Rounded corners.

Thin outlines.

Minimal strokes.

Consistent corner radius.

Everything should feel geometric.

---

# Characters

Characters are optional.

When included they should be:

cute

cartoon

small

non-human

helper spirits

tiny explorers

astronomers

friendly robots

The product is the hero.

---

# Environment

Preferred environments:

night sky

mountains

observatory

space

minimal clouds

northern lights

deep blue gradients

Never use:

cities

offices

people

abstract blobs

floating cubes

---

# Workflow Diagrams

Workflow diagrams should resemble constellations.

Nodes glow.

Connections are thin.

Animations (if applicable) move like starlight.

Avoid:

flowchart boxes

heavy arrows

Visio style diagrams

---

# Logo Rules

The logo should always be:

simple

memorable

vector

usable at 32x32

The shield icon is the primary brand mark.

The Orion constellation should subtly influence its geometry.

Do not redraw the logo differently for every image.

---

# Typography

Modern geometric sans-serif.

Heavy headings.

Large spacing.

Minimal text.

Never decorate typography.

---

# Banners

GitHub banners should:

feel cinematic

tell a story

contain negative space

remain readable

avoid clutter

The logo should always have room to breathe.

---

# Screenshots

When generating UI mockups:

Use dark mode.

Rounded panels.

Soft shadows.

Purple highlights.

Consistent spacing.

Never create impossible UI.

Everything should feel buildable.

# Consistency Rule

Every new asset should look like it belongs beside the existing ones.

If two images are placed next to each other on the README, they should clearly appear to come from the same artist.

When unsure:

Prefer simplicity over detail.

Prefer whitespace over clutter.

Prefer elegance over realism.
